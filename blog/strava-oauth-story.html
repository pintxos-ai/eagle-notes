串接 Strava 有多難？
一個人摸索與有教練指導的差別

那些我們踩過的坑，以及有人陪的重要性

說起串接 Strava API這件事，很多人可能會想：「不就是拿個 API Key，照著文件寫程式嗎？」

嗯......如果真這麼簡單就好了。

我們實際碰到的困難

🔴 問題一：redirect_uri 一直錯

第一次串接的時候，我們信心滿滿地設定好 Client ID 和 Secret，信心滿滿地按下「授權」，然後——

「Invalid redirect_uri」

蛤？什麼東西？我明明就有設定啊！

後來才發現，Strava 的 redirect_uri 必須完全匹配，連一個斜線都不能多或少。那時候我們在本地測試用 localhost:3000/callback，上線後改成網域，結果就這樣卡了整整兩天。

🔴 問題二：Token 過期才知道

好了，授權成功了，資料也抓到了，我們開心地把系統上線。兩個星期後——

奇怪，怎麼沒有新的運動資料了？打開日誌一看，全部都是 401 Unauthorized。

原來 Strava 的 access token 只有 6 小時壽命，必須靠 refresh token 去更新。但文件寫得不清不楚，我們又沒經驗，就這樣傻傻地讓 token 過期、使用者資料全部停止同步。

🔴 問題三：瀏覽器環境讓你懷疑人生

有的時候，問題根本不在你的程式碼，而是——

「為什麼在本機可以，在正式環境不行？」

CORS 問題、Cookie 被擋、第三方Cookie政策⋯⋯每次部署都像在抽獎。你永遠不知道哪個瀏覽器會突然給你出一題。

🔴 問題四：文件看了還是霧煞煞

Strava 的 API 文件老實說不算差，但對於沒有串過 OAuth 的人来说，那些術語——Authorization Code、Access Token、Refresh Token、Scope——簡直像在看天書。

我們花了整整一個禮拜，才搞懂「為什麼要先拿授權碼再去換 token」這個流程。

如果當時有教練，會怎麼樣？

老實說，以上這些問題，全部都是「有人踩過」的坑。如果你一開始就有人帶，結果會完全不同：

🛡️
幫你避坑
「redirect_uri 要完全match，本地跟正式環境要分開設定。」一句話就省你兩天。
⏰
省時間
不用自己Google、翻論壇、看Stack Overflow，直接得到正確答案。
⚡
即時解答
遇到問題馬上問，不用卡關卡半天，不知道要找誰幫忙。
🧭
給你方向
不只是解決當下問題，還告訴你「為什麼會這樣」和「下次要注意什麼」。

教練的價值，不只是技術

很多人以為請教練就是「有人幫你寫Code」，其實不是。

教練真正厲害的地方在於：

  • 經驗傳承 — 他們走過的路比我們多，知道哪裡會摔跤
  • 節省時間 — 我們自己摸索兩週的問題，他們可能五分鐘就搞定
  • 心理支持 — 當你對著電腦抓狂三小時，有個人告訴你「這很正常，我也碰過」，那種舒壓感是無價的
  • 長期視角 — 教練不只幫你解決當下問題，還會幫你想「這功能一年後會不會有問題？」
