你有沒有想過，當你打開 Strava 看見今天的跑步距離、配速、心率……這些數據是怎麼「跑」到你的手機上的？

一、串接 Strava API？聽起來不難啊

一開始我們也是這麼想的。Strava 有完整的 API 文件，照著做不就好了？

結果證明，我們太天真了。

1. OAuth 流程：不是只有按一個按鈕那麼簡單

Strava 使用的是 OAuth 2.0 認證機制。簡單來說，流程是這樣的：

  1. 使用者點擊「用 Strava 登入」
  2. 跳轉到 Strava 的授權頁面，使用者同意讓我們存取數據
  3. Strava 會把使用者導回我們指定的 redirect_uri，並夾帶一個 code
  4. 我們用這個 code 去換取 access_token
  5. 最後，用這個 token 來叫用 API 取得運動數據

聽起來很標準？但問題才剛開始。

2. redirect_uri 的地雷

Strava 的 API 對 redirect_uri 有嚴格的要求——必須完全符合你在 Strava 開發者後台註冊的 URL，連一個斜線都不能差。

我們一開始在本地端測試，用的是 http://localhost:3000/callback，結果部署到正式環境後，换成 https://eagle-notes.com/callback，就這樣炸了。

錯誤訊息： Redirect URI mismatch

後來我們學乖了，把所有可能的環境（本地、測試、生產）都註冊到 Strava 的應用程式設定裡。解決方法不難，但過程真的很折騰。

3. Token 過期與重新整理

Strava 的 access_token 有 6 小時的有效期。6 小時後，你的系統就會發現：唉怎麼資料拿不到了？

這時候你需要 refresh_token 來刷新。但麻煩的是，每次刷新都會給你一組新的 token，舊的直接失效。如果你的系統有多個地方在用 token，很容易出現「A 刷新後 B 失效」的Race Condition。

我們後來做了一個 Token Manager，統一管理刷新邏輯，確保所有模組都用同一組有效的 token。這件事不難，但需要經驗——沒踩過坑不會知道。

二、防爬蟲？沒有瀏覽器？這些才是大魔王

如果你以為 API 串接就是最難的部分，那麼接下來的問題會讓你重新思考人生。

1. Strava 的防爬蟲機制

Strava 並不反對開發者使用 API，但他們對於異常流量非常敏感。如果你短時間內發送太多請求、從來不緩存資料、或是行為模式看起來像機器人——對不起，你的 API Key 很可能會被暫時封鎖。

我們曾經做過一個「每分鐘同步一次」的自動化任務，結果跑沒多久就被擋下來了。後來學會了：

  • 實作指數退避（Exponential Backoff）：被擋就等一下，然後再試，時間拉長
  • 善用 Webhook： Strava 有事件通知機制，運動結束後主動推給我們，不用一直 polling
  • 本地緩存 + 差量更新：只拿有變化的資料，不要每次都全文下載

2. 沒有瀏覽器環境的困境

這點可能是最容易被忽略的問題。

很多運動 App（不只是 Strava）以為使用者都會用手機或電腦操作。但當你想在伺服器上定時抓資料、做一些自動化分析時，你會發現——很多 API 壓根沒有考慮過「非瀏覽器」的場景。

比如說，有些 API 要求你必須有有效的 session、必須通過 JavaScript 渲染的頁面驗證、或是需要操作瀏覽器 cookie。這些在人類眼裡「很正常」的操作，放到伺服器端就變成了技術障礙。

我們後來用了 Puppeteer 來模擬瀏覽器行為，才解決了某些需要「人為操作」的環節。代價是：伺服器資源變多、維護變複雜、速度變慢。

「本來只是想抓個數據，結果需要養一台無頭瀏覽器。」——某工程師的心聲

三、這些困難告訴我們的事：AI 教練不是只會說「加油」

好，講了這麼多技術細節，你可能會問：所以呢？

所以我們從這些折騰中，體悟到了一個很重要的道理：一個好的 AI 教練，不能只是給建議。

1. 沒有數據，AI 什麼都不是

你叫 AI 幫你分析訓練成效，但如果你沒有可靠的數據來源——那些建議頂多只是「多休息」、「跑慢一點」之類的廢話。

真正的 AI 教練需要：

  • 穩定的數據來源：不只是 Strava，可能是 Apple Health、Google Fit、Garmin……每個來源的格式都不一樣，你需要一個翻譯層
  • 即時的同步能力：如果你跑完步還要手動輸入數據，那跟寫日記有什麼兩樣？
  • 數據品質的把關：GPS 漂移、心率異常、設備没同步……AI 需要懂得過濾雜訊

這就是為什麼 Eagle Notes 願意花這麼多時間在「串接」這件事上。我們知道，最好的建議來自於最真實的數據。

2. 自動化才是關鍵

假設你今天有一個完美的 AI 教練，它分析你的訓練、給你客製化的建議……但如果你每次都要自己把數據複製貼上，那請問這個「教練」跟 Google 有什麼兩樣？

自動化才是讓 AI 教練發揮價值的關鍵：

  • 運動結束 → 自動同步 → AI 分析 → 訓練建議
  • 每週數據趨勢 → 自動產出報告 → 調整下週目標
  • 異常警訊（恢復時間不足、訓練過度）→ 主動通知

這整個流程，需要的是技術堆疊——從 API 串接、資料庫儲存、自動化排程、到 AI 推理——每一層都不能有短板。

這也是 Eagle Notes 正在做的事。我們不只想做一個「會說話的訓練計劃」，我們想做的，是一個會自己幫你搞定一切的 AI 教練。
